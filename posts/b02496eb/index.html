JVM

功能：解释和运行 内存管理 即时编译JIT

虚拟机组成

类加载器、运行时数据区域、执行引擎（即时编译、解释器，垃圾回收）、本地接口（c、c++）

字节码：基础信息，常量池，字段，方法，属性

类的生命周期
生命周期：加载-》内存，连接-》验证；准备；解析，初始化-》为静态方法赋值，使用，销毁

类的初始化方法：4个

实例化对象的5种方法//1.new//class.forName//构造方法//克隆//序列化的反序列化

静态代码块>构造代码块>构造方法

类加载器：3个：

启动类加载器，扩展类加载器，应用程序类加载器

双亲委派机制：
1.保证类加载的安全性
2.避免重复加载
当一个类需要加载时，向上查找是否加载过-向下委派查看是否能加载。

运行时数据区：

线程不共享 线程共享
程序计数器 java虚拟机栈 本地方法栈 方法区 堆

方法区：存放类的元信息，运行时常量池，字符串常量池

程序计数器 java虚拟机栈 本地方法栈：线程不共享。伴随线程的创建而创建，销毁而销毁，不需要垃圾收集器管理。
垃圾回收器：回收堆内存

方法区的回收：3个条件
1.所有实例对象都被回收，在堆中不存在任何该类的实例对象以及子类对象
2.加载该类的类加载器已经被回收
3.该类对应的java.lang.Class对象没有在任何地方被引用。

System.gc() //向垃圾回收器发消息，不一定会被回收

如何判断堆上的对象可以回收？根据对象是否被引用
循环依赖

堆回收：

引用计数法
引用+1，取消-1

可达性分析器：GC Root 通过引用链可达
GC Root :
1.线程Thread对象，应用线程栈帧中的方法参数、局部变量等。
2.系统类加载器加载的java.lang.Class对象
3.监视器对象，用来保存同步锁synchronized关键字持有的对象。
4.本地方法调用时使用的全局对象。

5种对象应用：

强引用：我们常常 new 出来的对象就是强引用类型，只要强引用存在，垃圾回收器将永远不会回收被引用的对象，哪怕内存不足的时候
软引用：使用 SoftReference 修饰的对象被称为软引用，软引用指向的对象在内存要溢出的时候被回收
弱引用：使用 WeakReference 修饰的对象被称为弱引用，只要发生垃圾回收，若这个对象只被弱引用指向，那么就会被回收
虚引用：虚引用是最弱的引用，在 Java 中使用 PhantomReference 进行定义。虚引用中唯一的作用就是用队列接收对象即将死亡的通知

强引用、软应用、弱引用、虚引用、终结器引用

垃圾回收算法：

标记-清除算法 复制算法 标记-整理算法 分带GC

分带GC：年轻代/老年代

年轻代：伊甸园区、幸存区1、幸存区2

常用的GC回收器：

Serial + serialOld
ParNew + CMS
PS + PO
G1

日志

一、日志功能的核心原理

日志的本质是将程序运行时的关键信息（时间、级别、内容、上下文等）持久化输出（输出到控制台、文件、数据库、远程服务器等），核心目的是：

  1. 记录程序运行状态，辅助调试和问题定位；
  2. 留存操作记录，满足审计、合规要求；
  3. 监控程序健康状态。

日志的核心组成要素

要素 作用
日志级别 区分信息重要程度（DEBUG/INFO/WARN/ERROR/FATAL），方便过滤关键信息
时间戳 记录日志产生的时间，是问题定位的核心维度
上下文信息 如模块名、线程 ID、用户 ID 等，补充日志的场景信息
输出目的地 控制台（stdout/stderr）、文件、ELK/MySQL 等，可配置多目的地输出
格式化规则 定义日志的展示格式（如 [时间] [级别] [模块] 内容）
