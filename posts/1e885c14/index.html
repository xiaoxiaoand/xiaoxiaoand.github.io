Redis

过期时间，redis监听，过期策略，

redis过期时间策略

惰性删除（Lazy）
用到 key 时才检查是否过期
不查不用，不删除
优点：省 CPU
缺点：占内存

定期删除（Periodic）
Redis 每秒抽查一批 key
发现过期就删除
平衡 CPU 和内存

内存淘汰机制（Eviction）
内存满了才删
比如：allkeys-lru（删最少使用的）

Redis 过期事件 不是实时、不保证可靠：
过期 key 可能延迟 1~5 秒才触发
高并发下可能丢失事件
不能做订单超时、支付超时这种严格业务

看门狗

Redis 分布式锁的自动续期机制

业务没执行完 → 锁快过期了
看门狗自动给锁延长过期时间
防止业务未完成，锁就失效

原理（Redisson 实现）
加锁时默认 30 秒过期
启动一个后台线程（看门狗）
每 10 秒检查锁是否还持有
是 → 重新设置为 30 秒
