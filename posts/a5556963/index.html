存储方式 存储位置 优点 缺点 适用场景
localStorage 浏览器本地（永久存储，除非手动删） 刷新 / 关闭页面不丢失； 容量大（5MB）； 操作简单 有 XSS 攻击风险； 不会自动过期 主流方案（配合过期时间控制）
sessionStorage 浏览器会话（关闭页面就清空） 安全性稍高（会话结束即失效）； 操作简单 刷新页面还在，但关闭页面就没； XSS 风险仍存在 临时登录（如公共电脑）
Cookie 浏览器 Cookie（可设置过期） 可配置 HttpOnly（防 XSS）； 自动随请求发送 容量小（4KB）； 有 CSRF 风险； 配置复杂 高安全要求场景（需后端配合）
Vuex/Pinia 内存（页面刷新就丢失） 响应式，方便组件使用； 无安全风险 刷新 / 关闭页面就没，需配合 localStorage 持久化 配合 localStorage 使用（内存 + 持久化）

localStorage + Pinia/Vuex 组合：

1.持久化存储（localStorage）：登录成功后把 token 存到 localStorage，保证刷新 / 重启页面不丢失；

2.内存存储（Pinia/Vuex）：把 token 同步到状态管理库，方便组件快速获取（响应式）；

3.请求拦截器携带 token：每次请求自动把 token 加到请求头，传给后端鉴权；
